Requisite Bugs
According to the OHRRPGCE FMF's license, the program must always contain at least one bug. This bug need not be mentioned (or even known) but it must be there. As it happens, each version of the OHRRPGCEFMF has traditionally hallmarked one bug as the most notorious bug for this version. This bug is then fixed for the next release. Here is the current bug history.
Picture Version of the FMF Bug Name Bug Type Description
Alpha: Five Percent Seth Can't Read Bohr Bug Due to a misread of "max tilesets in TIL" as "number of tilesets in TIL", the last tileset in any game was never loaded. Crescent Dream would crash because of this.
Alpha: Ghost Town The Energizer Tile Anim Heisenbug After one loop, a tile animation would not reset to the initial tile. Usually, developers did this automatically, but for some animations (like the teleporter in Wandering Hamster) this functionality was relied upon. This bug only appeared on the hardware; testing in an emulator just rendered the animation tile as a white square.
Alpha: Amphitheatre Hit & Run Dodo Bug Vehicles had a peculiar penchant for incorporating hero graphics in with their own. This bug up and vanished when an unrelated issue (NPCs walking too slowly) was fixed.

Version History
Each version of the OHRRPGCEFMF is special; here's why:
Screenshot Version Name Load Time* Notable Features Comments
Alpha: Five Percent ????
  • Loaded all simple map tiles; user could explore them by hitting "5"
  • Caching of tilesets was implemented even at this early stage (although it actually provided no performance gain, due to large .RPG file sizes.
  • Main menu & extensible format for drag-and-drop adding of RPGs to the GAME_FMF.JAR file
 This version took about two weeks for me to patch together, and I was immensely proud of it. Most people were non-plussed.
Alpha: Ghost Town 15 minutes
  • Tile-based movement for one hero
  • Ability to switch heroes with "1" key
  • Passability enabled. Overhead tiles enabled. Doors enabled.
  • A loading bar was extended to show which lump/piece of a lump was currently being parsed.
  • Walkabout graphics cached, obviously. They also pre-loaded, which did save time.
 This was when I knew I was doing something right: waiting a hideous quarter of an hour actually gave one the chance to experience what this platform had to offer. It felt like an OHR game when played on a real phone. There just... wasn't much to do yet. Hence the version name.
Alpha: Amphitheatre 12 seconds
  • Introduction of XRPG format; loading time dramatically reduced. Selected lumps compressed by roughly 79%
  • Tilesets and backdrops saved as PNGs, which was more fitting to the mobile format
  • For impatient players, the hero (and all NPCs) can be sped up by any increment of the walking speed.
  • NPCs, Message Boxes, Vehicles, and more!
 Don't let the list's brevity sway your opinion: switching to the XRPG format was a difficult decision, and supporting it took every trick in the book (Huffman Coding, faux-Huffman Coding, assumption-based aggregation). Yet to see the map actually pop up before my phone darkened the display was satisfying. The other features were nice, too.
Alpha: Prima Facie 20 seconds
  • The Menu
  • Pushable NPCs and chained message boxes
  • Player is informed of random encounters.
  • Games menu threads loading games list. Also, the "back" button on phones should work in the menu now.
 I'm quite happy with the menu, though it's a bit heavy. The encounters, strange as it may sound, add a huge boost to the feel that one's actually playing an RPG. Checking for bugs on the ride home, I could actually immerse myself in the tiny town of Flannat.
*On hardware

Comments
There's not much to say... enjoy the current release because who knows when adding another feature will be too much for a simple phone and production'll grind to a halt. Compare my top-of-the-line desktop with my top-of-the-line phone in terms of resources, and you might encourage me to give up right now. But I'm nothing if not temerarious.
